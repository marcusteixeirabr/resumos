### O que é um Banco de Dados?

Um **banco de dados** é uma coleção de dados persistentes que podem ser compartilhados e estão inter-relacionados. Mais do que um simples arquivo ou planilha, um banco de dados organiza fatos sobre pessoas, coisas e acontecimentos de forma que possam ser recuperados e mantidos com eficiência por múltiplos usuários e aplicações simultaneamente.

Você interage com bancos de dados todos os dias: ao passar o cartão no supermercado, ao sacar em um caixa eletrônico, ao se matricular em um curso ou ao encomendar um produto online. Cada uma dessas operações consulta e atualiza registros em bancos de dados computadorizados.

---

### Características Essenciais de um Banco de Dados

Todo banco de dados de negócio possui três propriedades fundamentais:

#### Persistência de Dados

Dados **persistentes** são armazenados em mídia não volátil (disco magnético, SSD, memória flash) e sobrevivem ao encerramento do programa que os criou. Uma variável em memória RAM desaparece quando o programa termina — um banco de dados não. A persistência não significa eternidade: dados obsoletos são removidos ou arquivados quando perdem relevância.

#### Acesso Compartilhado

Um banco de dados pode ter **múltiplos usuários e aplicações** acessando a mesma base ao mesmo tempo. Por exemplo, um banco de dados de pessoal pode suportar simultaneamente o cálculo de folha de pagamento, avaliações de desempenho e relatórios para o governo. O sistema garante que dois usuários não corrompam os mesmos dados ao tentar alterá-los simultaneamente.

#### Inter-relacionamento

Dados armazenados como **unidades separadas (entidades)** são conectados por relacionamentos, fornecendo uma visão completa. Uma entidade pode representar pessoa, lugar, coisa ou evento. Um banco de dados típico de negócio possui centenas de entidades e relacionamentos — por exemplo, ligar dados de *cliente* (nome, endereço) com dados de *pedido* (número, data) para processar vendas.

### Exemplos de Bancos de Dados de Negócio

#### Banco de Dados de uma Universidade

Entidades: alunos, professores, cursos, ofertas de cursos, matrículas.

Relacionamentos típicos:
- Quais ofertas estão disponíveis para um curso em um dado período?
- Quem é o instrutor de um curso?
- Quais alunos estão matriculados em uma oferta?

#### Banco de Dados de uma Estação de Água

Entidades: clientes, metros cúbicos consumidos, contas, pagamentos, leituras de relógio.

O banco de dados controla o consumo periódico de cada consumidor, calcula a conta levando em conta histórico de pagamento e tipo de consumidor, e registra início e interrupção do serviço.

#### Banco de Dados de um Hospital

Entidades: pacientes, profissionais da saúde, diagnósticos, tratamentos, sintomas.

Múltiplos profissionais leem e atualizam o mesmo prontuário: médicos prescrevem tratamentos, enfermeiras registram sintomas, nutricionistas planejam dietas. O banco de dados garante que todas essas informações coexistam sem conflito.

### Impacto na Tomada de Decisão

Bancos de dados não melhoram apenas as operações do dia a dia — eles sustentam decisões estratégicas de longo prazo: onde instalar uma nova loja, em quais equipamentos investir, quais produtos adicionar ao estoque. A tecnologia de banco de dados transforma grandes volumes de fatos brutos em informações acionáveis para gestores.

### Evolução Histórica

| Período | Marco |
|---|---|
| **1970** | Edgar Codd propõe o **Modelo Relacional** na IBM, base dos SGBDs modernos |
| **Anos 1980** | **SQL** consolida-se como linguagem padrão de consulta e manipulação de dados |
| **Anos 1990–2000** | SGBDs chegam à Internet; acesso via navegador torna os dados amplamente acessíveis |

### Práticas Modernas

- **Cloud databases (DBaaS)** — Amazon RDS, Azure SQL Database e Google Cloud SQL eliminam a necessidade de instalar e administrar servidores de banco de dados. Você cria uma instância em minutos e paga pelo uso.
- **PostgreSQL** — o SGBD relacional open-source mais avançado atualmente, com suporte nativo a JSON, arrays, full-text search e extensões como `pgvector` para IA.
- **NoSQL** — para dados sem esquema fixo: MongoDB (documentos JSON), Redis (chave-valor em memória), Cassandra (colunares para escrita massiva) e Neo4j (grafos).
- **NewSQL** — combina a consistência do modelo relacional com a escalabilidade horizontal do NoSQL: CockroachDB, PlanetScale, Google Spanner.
- **Bancos vetoriais** — armazenam embeddings gerados por modelos de linguagem (LLMs) e permitem busca por similaridade semântica: Pinecone, Weaviate, pgvector.
- **Data warehouses modernos** — Snowflake, BigQuery e Redshift substituíram os data warehouses tradicionais: armazenamento colunar, escalonamento automático e separação de compute/storage.
- **ACID ainda importa** — as propriedades de integridade transacional (Atomicidade, Consistência, Isolamento, Durabilidade) permanecem o padrão de ouro para sistemas financeiros e de missão crítica.

<Callout type="tip">
**Dicas para a Prova**

- As três características essenciais de um banco de dados são **persistência**, **compartilhamento** e **inter-relacionamento** — decorar essa tríade resolve a maioria das questões conceituais.
- **Persistência** = armazenamento não volátil que sobrevive ao fim do programa (resposta da questão 2: opção B).
- **Inter-relacionamento** ocorre quando entidades *diferentes* são ligadas — ex.: cliente × número do pedido (questão 3, opção A). Dados de uma mesma entidade (nome × CPF do cliente) não constituem inter-relacionamento.
- Uma **transação** é uma unidade de trabalho que deve ser processada de forma confiável, sem interferência de outros usuários e sem perda de dados por falhas (questão 4, opção C).
- Informações úteis para e-commerce: frequência de compras, valor das compras, produtos adquiridos e forma de pagamento — *não* incluem dados pessoais irrelevantes como estado civil ou data de nascimento (questão 5, opção E).
- Ler um capítulo de livro **não** envolve tecnologia de banco de dados computadorizado (questão 1, opção D).
</Callout>
