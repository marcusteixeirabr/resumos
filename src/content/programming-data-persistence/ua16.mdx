### O que é um framework?

Um **framework** é uma estrutura de suporte para construir software — uma base sólida projetada e testada por outros desenvolvedores para que você não precise começar do zero.

**Por que usar frameworks?**

- Economiza tempo e reduz o risco de erros de implementação
- Código normalmente mais limpo, testado e confiável
- Evita retrabalho e código duplicado
- Permite que o desenvolvedor se concentre na lógica de negócio específica do projeto
- Pode ser estendido com outras bibliotecas, módulos e frameworks

**Framework vs. Biblioteca — inversão de controle:**

| Conceito | Quem está no controle |
|---|---|
| **Biblioteca** | *Seu código* chama a biblioteca para resolver uma questão específica |
| **Framework** | *O framework* chama seu código — você preenche os "buracos" definidos pelo framework |

Essa diferença é chamada de **inversão de controle**: ao usar um framework, você não chama o framework — o framework chama você.

**Tipos comuns de frameworks:**

| Tipo | Exemplos |
|---|---|
| **Web** | Django (Python), Rails (Ruby) |
| **Mobile** | Flutter, Xamarin, React Native, Ionic |
| **Persistência** | Hibernate, EclipseLink, Torque |
| **Gerenciamento de dependências** | Maven, Gradle |

### Hibernate — framework ORM para Java

O **Hibernate** é um framework de **ORM** (Object-Relational Mapping) que resolve o problema de persistir objetos Java em bancos de dados relacionais.

O desafio: aplicações Java trabalham com **objetos** (atributos + comportamentos), mas bancos relacionais armazenam **tabelas** (linhas + colunas). Traduzir entre esses dois mundos — o chamado **mismatch objeto-relacional** — é trabalhoso com JDBC puro, exigindo código de estrutura repetitivo (*boilerplate*).

O Hibernate faz esse mapeamento automaticamente:

```
Classe Java (POJO)  →  Tabela do banco
Atributo da classe  →  Coluna da tabela
Objeto (instância)  →  Linha/registro
```

O Hibernate **implementa a especificação JPA** (Java Persistence API), que é o padrão oficial Java para persistência. JPA é uma *especificação* — o Hibernate é uma *implementação*.

**Vantagens do Hibernate:**

| Vantagem | Detalhe |
|---|---|
| **Código aberto** | Licença LGPL (Lesser General Public License) |
| **Alto desempenho** | Cache interno — consultas frequentes são armazenadas em memória |
| **HQL independente** | Hibernate Query Language (HQL) gera SQL compatível com qualquer banco — sem precisar trocar queries ao mudar de SGBD |
| **Criação automática de tabelas** | Com `hbm2ddl.auto`, o Hibernate cria ou atualiza o schema automaticamente |
| **JOINs simplificados** | Buscar dados de múltiplas tabelas é transparente via mapeamentos de relacionamento |
| **Estatísticas** | Fornece métricas de consultas e status do banco de dados |

**Dependências internas do Hibernate:**

- **JDBC** — camada de acesso ao banco de dados relacional
- **JTA** (Java Transaction API) — gerenciamento de transações
- **JNDI** (Java Naming and Directory Interface) — lookup de recursos em servidores de aplicação

### Configuração do Hibernate (hibernate.cfg.xml)

O Hibernate é configurado por um arquivo XML chamado `hibernate.cfg.xml`. Ele define a conexão com o banco, o driver, o dialeto SQL e o mapeamento das classes.

```xml
<hibernate-configuration>
  <session-factory>
    <!-- Conexão com o banco -->
    <property name="connection.url">
      jdbc:mysql://localhost:3306/BD
    </property>
    <property name="connection.driver_class">
      com.mysql.jdbc.Driver
    </property>
    <property name="connection.username">usuario</property>
    <property name="connection.password">senha</property>

    <!-- Dialeto SQL do banco -->
    <property name="dialect">
      org.hibernate.dialect.MySQL5Dialect
    </property>

    <!-- Mapeamento da classe para a tabela -->
    <hibernate-mapping>
      <class name="pacote.Classe" table="NOME_TABELA">
        <id name="id" column="ID">
          <generator class="native"/>
        </id>
        <property name="atributo1" column="COLUNA1"/>
        <property name="atributo2" column="COLUNA2"/>
      </class>
    </hibernate-mapping>
  </session-factory>
</hibernate-configuration>
```

**Elementos principais do arquivo de configuração:**

| Elemento | O que configura |
|---|---|
| `connection.url` | URL JDBC de conexão com o banco |
| `connection.driver_class` | Driver JDBC a usar |
| `connection.username` / `password` | Credenciais de acesso |
| `dialect` | Variante SQL do banco (MySQL5Dialect, PostgreSQLDialect, etc.) |
| `<class>` | Mapeamento classe Java ↔ tabela do banco |
| `<id>` | Atributo mapeado como chave primária |
| `<generator class="native">` | Geração automática da PK pelo banco |
| `<property>` | Atributos mapeados como colunas |

> Se o XML não for fornecido, o Hibernate lê as **anotações JPA** diretamente nas classes Java — abordagem preferida atualmente.

### Anotações JPA no Hibernate

Em vez de XML, as anotações definem o mapeamento diretamente na classe:

```java
@Entity
@Table(name = "cliente")
public class Cliente {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private int id;

    @Column(name = "nome", nullable = false)
    private String nome;

    @Column(name = "email")
    private String email;

    // getters e setters...
}
```

| Anotação | Função |
|---|---|
| `@Entity` | Marca a classe como entidade persistível (mapeia para uma tabela) |
| `@Table(name="...")` | Define o nome da tabela (opcional — usa o nome da classe se omitido) |
| `@Id` | Marca o atributo como chave primária |
| `@GeneratedValue` | Define a estratégia de geração da PK (IDENTITY, SEQUENCE, AUTO) |
| `@Column` | Configura o mapeamento da coluna (nome, nullable, length, etc.) |

### Persistindo objetos com Hibernate

O fluxo de persistência usa a cadeia: `StandardServiceRegistry` → `Metadata` → `SessionFactory` → `Session` → `Transaction`.

```java
import org.hibernate.Session;
import org.hibernate.SessionFactory;
import org.hibernate.Transaction;
import org.hibernate.boot.Metadata;
import org.hibernate.boot.MetadataSources;
import org.hibernate.boot.registry.StandardServiceRegistry;
import org.hibernate.boot.registry.StandardServiceRegistryBuilder;

public class ArmazenarDados {
    public static void main(String[] args) {

        // 1. Carrega a configuração do hibernate.cfg.xml
        StandardServiceRegistry ssr = new StandardServiceRegistryBuilder()
            .configure("hibernate.cfg.xml").build();

        // 2. Cria a SessionFactory (uma por aplicação — custo alto)
        Metadata meta = new MetadataSources(ssr).getMetadataBuilder().build();
        SessionFactory factory = meta.getSessionFactoryBuilder().build();

        // 3. Abre uma Session (uma por operação — custo baixo)
        Session session = factory.openSession();

        // 4. Inicia a transação
        Transaction t = session.beginTransaction();

        // 5. Cria e persiste o objeto
        Cliente cliente = new Cliente();
        cliente.setNome("João Silva");
        session.save(cliente);

        // 6. Confirma (commit) e fecha
        t.commit();
        session.close();
        factory.close();
    }
}
```

**Responsabilidades de cada componente:**

| Componente | Responsabilidade |
|---|---|
| `StandardServiceRegistry` | Carrega configurações do arquivo XML |
| `SessionFactory` | Criada uma vez por aplicação; cara para criar — mantida durante todo o ciclo de vida |
| `Session` | Uma por unidade de trabalho (request ou operação); mapeia para uma conexão JDBC |
| `Transaction` | Delimita a unidade atômica de trabalho (`beginTransaction()` / `commit()` / `rollback()`) |
| `session.save()` | Persiste o objeto no banco (equivale a INSERT) |
| `t.commit()` | Confirma as mudanças no banco |

<Callout type="warning">
**Material desatualizado — leitura crítica necessária**

- `com.mysql.jdbc.Driver` está **deprecated** desde MySQL Connector/J 5.x — use `com.mysql.cj.jdbc.Driver`.
- `MySQL5Dialect` está **deprecated** nas versões recentes do Hibernate — use `MySQLDialect` ou especifique a versão correta.
- O livro não usa **try-with-resources** para fechar `Session` e `SessionFactory` — em produção, sessions não fechadas causam vazamento de conexões.
- Credenciais hardcoded no `hibernate.cfg.xml` são um **risco de segurança** — devem vir de variáveis de ambiente ou ferramentas como Vault.
- O livro não menciona **pool de conexões** — sem pool, cada `session.openSession()` abre uma nova conexão JDBC, inviável em produção.
- `session.save()` foi **deprecated** no Hibernate 6 — use `session.persist()`.
- AngularJS (citado nos tipos de frameworks web) foi **descontinuado** em 2021 — o framework atual é o **Angular** (totalmente diferente).
- Sem JPA 3.x: hoje usa-se `jakarta.persistence.*` em vez de `javax.persistence.*` (mudança do Jakarta EE 9+).
</Callout>

### Práticas Modernas

**Spring Boot + Spring Data JPA — a forma padrão da indústria:**

Em vez de configurar `hibernate.cfg.xml` manualmente, o ecossistema Spring Boot auto-configura tudo via `application.properties`:

```properties
# application.properties
spring.datasource.url=jdbc:mysql://localhost:3306/meubanco
spring.datasource.username=${DB_USER}
spring.datasource.password=${DB_PASS}
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver
spring.jpa.hibernate.ddl-auto=validate
spring.jpa.show-sql=false
```

```java
// Repository — Spring Data JPA gera o SQL automaticamente
@Repository
public interface ClienteRepository extends JpaRepository<Cliente, Long> {
    List<Cliente> findByNome(String nome);
    Optional<Cliente> findByEmail(String email);
}

// Service
@Service
@Transactional
public class ClienteService {
    @Autowired
    private ClienteRepository repo;

    public Cliente salvar(Cliente c) {
        return repo.save(c);
    }
}
```

**HikariCP — pool de conexões (padrão no Spring Boot):**

```properties
spring.datasource.hikari.maximum-pool-size=10
spring.datasource.hikari.minimum-idle=5
spring.datasource.hikari.connection-timeout=30000
```

**Migrations com Flyway:**

```properties
spring.flyway.enabled=true
spring.flyway.locations=classpath:db/migration
# Arquivo: resources/db/migration/V1__create_cliente.sql
```

**Jakarta Persistence 3.x (Jakarta EE 9+):**

```java
// Antes (javax — legado)
import javax.persistence.Entity;
import javax.persistence.Id;

// Agora (jakarta — padrão atual)
import jakarta.persistence.Entity;
import jakarta.persistence.Id;
```

**Quarkus com Hibernate Panache (alternativa moderna):**

```java
@Entity
public class Cliente extends PanacheEntity {
    public String nome;
    public String email;

    public static List<Cliente> findByNome(String nome) {
        return list("nome", nome);
    }
}
```

**Boas práticas atuais:**
- Nunca usar `hbm2ddl.auto=create-drop` em produção — use Flyway ou Liquibase para migrations versionadas
- Sempre usar pool de conexões (HikariCP é o padrão do Spring Boot)
- Credenciais via variáveis de ambiente ou Vault (nunca em arquivos de configuração versionados)
- Prefer `session.persist()` sobre `session.save()` (Hibernate 6+)
- Try-with-resources para `Session` em código legado sem Spring
- Usar `@Transactional` do Spring em vez de gerenciar `Transaction` manualmente

### Dicas para a Prova

<Callout type="tip">
**Q: O que é um framework e como diferencia de uma biblioteca?**
Framework: estrutura que chama seu código (inversão de controle). Biblioteca: seu código chama a biblioteca.

**Q: O que é ORM?**
Object-Relational Mapping — técnica de mapeamento entre objetos Java e tabelas relacionais. O Hibernate é um framework ORM.

**Q: O que é o Hibernate?**
Framework ORM open source para Java que implementa a especificação JPA — mapeia classes (POJOs) para tabelas do banco relacional automaticamente.

**Q: Qual a relação entre Hibernate e JPA?**
JPA é a *especificação* (define contratos e anotações). Hibernate é uma *implementação* concreta da JPA.

**Q: Para que serve o hibernate.cfg.xml?**
Arquivo de configuração XML que define a URL de conexão, driver, usuário, senha, dialeto SQL e o mapeamento das classes para tabelas.

**Q: Qual anotação JPA define uma classe como entidade (tabela)?**
`@Entity`. Para marcar a chave primária: `@Id`.

**Q: Qual é a hierarquia de objetos do Hibernate?**
`StandardServiceRegistry` → `SessionFactory` → `Session` → `Transaction`.

**Q: Qual objeto do Hibernate deve ser criado apenas uma vez por aplicação?**
`SessionFactory` — é caro de criar e deve ser mantido durante todo o ciclo de vida da aplicação.
